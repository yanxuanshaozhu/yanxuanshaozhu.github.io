Java Basic

  1. JDK, JVM, JRE

    • JDK > JRE > JVM
    • JDK is complete Java SDK, JDK = JRE + compiler(javac) and tools(javadoc)
    • JRE is the java runtime, cannot create java programs, JRE = JVM + libraries + other tools
    • JVM multi-platform virtual machine, has different implementations on different OSs

  2. Binary code: source program -> (javac compile) binary code -> (JIT) mathine code

    • Compiled language: source code compile to machine code then run, C, C++,...
    • Interpreted language: interpreter interprets code during runtime, Python, JavaScript

  3. Basic syntax

    • <<: left shift operator, x << 1 is equivalent to x * 2, lower positions are filled with 0s
    • >>: signed right shift, x >> 1 is equivalent to x * 2, if x isnegative, higher positions are filld with signs
    • >>> unsigned right shift, higher positions are filled with 0s
    • For overflow, the shift is rounded first, eg for int x, x << 42 is equivalent to x << 10 since the maximum amount of bits for an integer is 32

  4. Java primitive types

    • byte, short, int, long, float, double, char, boolean
    • Wrapper class can be used in generic, different stored place(heap vs stack), different comparison methods (== vs equals)
    • Cache for wrapper class: Byte, Short, Integer, Long caches [-128, 127], Character [0, 127], Boolean stores Ture/False, Double/Float does not have cache
    • Auto-boxing and auto-unboxing
    • Truncation error for floating point numbers, use BigDecimal to avoid the error
    • For large integer values, use BigInteger

  5. Instances of the same class share same static variables, so there is less memory usage

  6. Static methods

    • Can be invoked using Class.method or Instance.method
    • Can access only static methods/fields, cannto access instance methods/fields (initialization order: static field, static block, instance field, instance block, constructor)

  7. Overloading vs overriding

    • Overloading: same name, different argument type, order, amount, can have different return value or access modifier
    • Overriding: same name, same argument, same or narrower scope of return value and exception, larger scope of access modifier, constructor cannot be overridded

  8. Variable arguments in Java

    • Can have 0 or more arguments
    • Must be the last argument in a method

  9. Object oriented programming vs procedural programming

    • PP: step by step approach
    • OOP: focus on object
    • Example: calcualte the area of a circle

  10. OOP characteristics

    • Encapsulation: encapsulate information inside object, access data through API
    • Inheritance: child does have but cannot access private field/method in parent class, reduce code redundancy, increase readability
    • Polymorphism: compile time type != runtime type

  11. Interface vs abstract class

    • Common: cannot be instantiated, can have abstract method, can have concrete method(default method in interface in Java 8)
    • Difference: implement n interfaces, extend 1 class, interface fields are public static final

  12. hashCode and equals

    • Different hashCode => different objects
    • Same hashCode, different equals => different objects
    • Same hashCode, same equals => same object

  13. String, StringBuffer, StringBuilder

    • String is immutable(private final byte value[]; before Java 9 it's private final char value[];, this change was made for better encoding reason), StringBuilder and StringBuffer extends AbstractStringBuilderand they are mutable
    • String is immutable and so thread safe, StringBuffer uses synchronized for its method so is thread safe, StringBuilder is not thread safe
    • Before JDK 9, String concatenation using + actually calls append() in StringBuilder, using + in a loop is slow because it creates multiple StringBuilder instead of 1. After JDK 9 + uses makeConcatWithConstants() and there is no efficiency issue
    • String constant pool

  14. Exception and error

    • java.lang.Throwable
      • Exception: can be handled by program
        • Checked exception should be handled, otherwise there will a compile error. ClassNotFoundException, IO related exceptions, SQLException, ...
        • Unchecked exception (Runtime exception), NullPointerException, IllegalArgumentException, ...
      • Error: cannot be handled by program, generally program will stop
    • Finally is not always executed System.exit(1) stops the JVM
    • try-with-resources (for AutoCloseable or Closeable resources, like Scanner, Stream, Writer, ...)

  15. Generic

    • Introduced in JDK5, using generic parameters can increase readability and stability. Compiler checks generic argument compatibility
    • Generic parameter can be used on class, method or interface(when implementing a generic interface, you can either implement or do not implement the generic parameter)

  16. Reflection

    • Allow you to dynamically get all fields and methods of a class during runtime
    • Dynamic delegation and annotation uses reflection

  17. Annotation

    • Introduced in JDK5
    • Annotation actually extends the Annotation class

  18. Serialization and deserialization

    • Serialization: convert data into a format that can be persisted or transported, deserialization: convert serialized binary sequence back to data
    • Common usage: convert a stream into an object
    • The Object should implements the java.io.Serializable interface(recommended to provide a serialVersionUID)
    • transient keyword: if a field is marked as transient, then that field is not serialized, then during deserialization, the default value for that type will be used
    • Disadvantage of java built-int serialization approach
      • Not supported by other language
      • Poor performance: the serialized stream is large in size
      • Safety issue: the serialized stream can be manipulated during deserialization, which suffers from safety issues
    • Other serialization protocol: XML, JSON, Fastjson, Kryo, Avro...

  19. IO

    • Basics
      • Top level streams: InputStream/OutputStream(for byte stream), Reader/Writer(for character stream)
      • FileInputStream wrapped by BufferedInputStream or ObjectInputStream or DataInputStream, etc.
      • FileOutputStream wrapped by BufferedOutputStream or ObjectInputStream or DataOutputStream, etc.
      • InputStreamReader, FileReader, OutputStreamWriter, FileWriter
      • PrintStream, PrintWriter
    • BIO(blocking I/O), NIO(non-blocking/new I/O), AIO(asynchronous I/O, JDK7 NIO)

Java Collection

  1. Basics
    • Collection => List(ordered, can have duplicates), Set(unordered, unique values), Queue(ordered, can have duplicates), Map(k-v pairs)
    • Underlying data structures:
      • List: ArrayList(Object[] array DEFAULT_CAPACITY 10), LinkedList(doubly linked list: private static class Node<E>) are not thread safe, Vector(Object[] array) is thread safe
      • Set: HashSet(HashMap, unordered, unique), LinkedHashSet(LinkedHashMap), TreeMap(TreeMap, ordered, unique)
      • Queue: PriorityQueue(Object[] array based binary heap), ArrayDeque(Object[] array with the head and tail pointers)
      • Map: HashMap(Node<K,V>[] array, 0.75 default load factor, 8 TREEIFY_THRESHOLD, 6 UNTREEIFY_THRESHOLD, 64 MIN_TREEIFY_CAPACITY), LinkedHashMap(based on HashMap), Hashyable(Node<?,?> array + similiar to HashMap fields), TreeMap(Red-Black tree based map)
  2. Comparable and Comparator
    • If a class implements the java.lang.Comparable and overrides the public int compareTo(Object obj) method, you can use Collections.sort(list) to sort the list of that class objects
    • You can also use provide a java.util.Comparator to the sort method if the class does not implement the Comparable interface. This works for both Arrays.sort() and Collections.sort()
  3. Map
    • You can provide a Comparator to the constructor when creating a TreeMap to specify how the keys are ordered
    • Using entrySet to iterate through a map is faster than using the keySet
  4. Collections
    • sort, reverse, swap
    • synchronizedCollection, synchronizedList, synchronizedList, synchronizedSet
  5. Others
    • Using isEmpty is faster than size() == 0
    • Use remove method on the iterator
    • Collection.removeIf(filter) to remove items from collections according a specific condition: list.removeIf(item -> item % 2 == 0);
    • Convert array to List and vice versa
    • asList()
      • You cannot use array that contains primitive values, should use the wrapper objects
      • The result list does not support add/remove/clear
      • Approach to support add/remove/clear
      • Approach to solve both the problems

Concurrent Programming

  1. Basics
    • JVM is a process, each program has multiple threads, with at least the main thread included. Each process has its own memory space
    • A thread has its own stack and program counter but multiple stacks share the heap
    • Concurrency and parallelism
      • Concurrency: multiple tasks execute in turn on a single CPU core
      • Parallelism: multiple tasks execute at the same time on multiple CPU cores
  2. Threading
    • State: new, waiting, runnable, blocked, terminated, timed_waiting
    • Context switch
    • Deadlock
  3. Multi-threading in Java
